## Overview

The ScrapeGraphAI app for Zapier connects any Zap to ScrapeGraph's v2 API as native Zapier actions ‚Äî fetch pages, extract structured JSON, run web searches, kick off multi-page crawls, and schedule monitors. Pair it with Zapier's 7,000+ apps to wire scraping into Slack, Sheets, Notion, Airtable, HubSpot, or anything else.

## Connect ScrapeGraphAI

1. In any Zap, search for **ScrapeGraphAI** as an action and pick one ‚Äî for example **Scrape a URL**.
2. When prompted, click **Sign in** ‚Üí paste your `SGAI-APIKEY` from the [dashboard](https://scrapegraphai.com/dashboard).
3. Save the connection ‚Äî Zapier reuses it across every ScrapeGraphAI step in every Zap.

<Frame>
  <img src="/integrations/images/zapier/connection.png" alt="ScrapeGraphAI connection dialog in Zapier with API Key field" />
</Frame>

<Note>
Your API key is stored on Zapier's side and is sent in the `SGAI-APIKEY` header on each call. Rotate it from the [dashboard](https://scrapegraphai.com/dashboard) and update the connection if needed.
</Note>

## What's in the integration

<Frame>
  <img src="/integrations/images/zapier/actions.png" alt="Zapier action picker showing ScrapeGraphAI actions" />
</Frame>


| Action | What it does |
|---|---|
| **Scrape a URL** | Fetch a page in markdown or HTML ‚Äî single round-trip |
| **Extract Data From URL** | Run a natural-language prompt over a URL, raw HTML, or markdown ‚Äî optional JSON schema |
| **Search Web** | AI web search with inline content; optional rollup prompt across results |
| **Crawl a Website** | Start an async multi-page crawl from an entry URL ‚Äî returns a job ID |
| **Get Crawl Status** | Poll a crawl job by ID until it returns the `pages` array |
| **Get a Past Result** | Fetch any stored job result by `id` or `scrapeRefId` |
| **Create Monitor** | Schedule a recurring fetch on a cron with diff detection and optional webhook |
| **Get Monitor Activity** | Read recent ticks from a monitor (`changed`, `diffs`, `status`, `createdAt`) |

<Note>
Zapier action timeouts cap individual steps at 30‚Äì60 seconds (depending on your plan). For larger crawls, use **Crawl a Website** to start the job, then a **Delay** + **Get Crawl Status** to poll ‚Äî same async pattern as n8n.
</Note>

### Extract Data From URL

Send a URL (or raw HTML / markdown) to ScrapeGraph and get back structured JSON, driven by a natural-language prompt.

  <img src="/integrations/images/zapier/extract.png" alt="Extract Data From URL action configuration in Zapier" />

| Field | Description |
|---|---|
| Source | `URL`, `HTML`, or `Markdown` ‚Äî picks which input field is used |
| URL | Page to extract from (when Source = URL) |
| HTML | Raw HTML to extract from (when Source = HTML) |
| Markdown | Markdown to extract from (when Source = Markdown) |
| Prompt | Natural-language instruction, e.g. `Extract product name and price` |
| Schema | Optional JSON schema to enforce output shape |
| Mode | Extraction mode ‚Äî `Auto`, `Fast`, or `JS` |

### Search Web

Run a web search and get the top results back inline, optionally with AI extraction applied across them.

|---|---|

### Create Monitor

Schedule ScrapeGraph to fetch a URL on a recurring cron and detect changes between runs.

<Frame>
  <img src="/integrations/images/zapier/monitor-create.png" alt="Create Monitor action with URL, name, cron, format, HTML mode, and webhook URL fields" />
</Frame>

| Field | Description |
|---|---|
| URL | Page to watch |
| Monitor Name | Optional display name |
| Interval (Cron) | 5-field cron expression ‚Äî see table below |
| Format | Content format captured on each tick (`markdown` / `html` / `links` / `summary`) |
| HTML Mode | Rendering mode ‚Äî Normal, Reader, or Prune |
| Webhook URL | Optional URL to POST tick payloads to |

**Common cron expressions**

| Schedule | Cron |
|---|---|
| Every hour | `0 * * * *` |
| Every 6 hours | `0 */6 * * *` |
| Daily at 09:00 UTC | `0 9 * * *` |
| Weekly on Monday | `0 9 * * 1` |

---

### Get Monitor Activity

Fetch the latest activity ticks from an existing monitor.

<Frame>
  <img src="/integrations/images/zapier/monitor-activity.png" alt="Get Monitor Activity action with Monitor ID and Limit fields" />
</Frame>

| Field | Description |
|---|---|
| Monitor ID | The `id` returned by Create Monitor |
| Limit | Number of ticks to return (1‚Äì100, default 20) |

Returns a `ticks` array where each entry has `changed` (boolean), `diffs`, `status`, and `createdAt`.

## Example Zap: extract product data into Google Sheets

A daily Zap that pulls product data from a listing page and appends each product as a row in Google Sheets.

1. **Trigger** ‚Äî `Schedule by Zapier` ‚Üí Every day.
2. **Action 1** ‚Äî `ScrapeGraphAI ‚Üí Extract Data From URL`:
   - **Source:** `URL`
   - **URL:** the product listing page
   - **Prompt:** `Extract all products on the page with their name, price, rating, and number of reviews`
   - **Schema:**
     ```json
     {
       "type": "object",
       "properties": {
         "products": {
           "type": "array",
           "items": {
             "type": "object",
             "properties": {
               "name": {"type": "string"},
               "price": {"type": "string"},
               "rating": {"type": "number"},
               "reviews": {"type": "number"}
             }
           }
         }
       }
     }
     ```
3. **Action 2** ‚Äî `Looping by Zapier ‚Üí Loop From Line Items`, fed from the previous step's `products` array. Zapier runs the next action once per item.
4. **Action 3** ‚Äî `Google Sheets ‚Üí Create Spreadsheet Row`:
   - **Name** ‚Üí `{{loop.name}}`
   - **Price** ‚Üí `{{loop.price}}`
   - **Rating** ‚Üí `{{loop.rating}}`
   - **Reviews** ‚Üí `{{loop.reviews}}`

Result: every product on the page gets its own row.

## Patterns that carry over

| Pattern | Action(s) | Notes |
|---|---|---|
| One-shot fetch | Scrape a URL | Cheapest path ‚Äî markdown by default |
| Structured extraction | Extract Data From URL | JSON schema is optional but locks the shape |
| Multi-page archive | Crawl a Website + Get Crawl Status + Get a Past Result | Loop over `pages` from Get Crawl Status, feed `scrapeRefId` into Get a Past Result |
| Recurring fetch with diff | Create Monitor + Get Monitor Activity | Or wire `webhookUrl` to a Zapier Webhook trigger for instant deltas |
| AI search rollup | Search Web with Prompt | Single call replaces "search ‚Üí scrape each ‚Üí summarize" |

## Troubleshooting

- **Action times out on Crawl a Website** ‚Äî large crawls run longer than Zapier's per-action limit. Keep Crawl a Website as the start step, then add a **Delay** + **Get Crawl Status** to poll until `status` is `completed`.
- **Extract returns an empty `json`** ‚Äî sharpen the prompt, or pin the shape with a JSON Schema. Pages that need rendering may need `Mode: JS`.
- **Connection test fails** ‚Äî confirm the API key is from the v2 dashboard (`scrapegraphai.com/dashboard`). v1 keys won't validate against the v2 API.
- **Get Past Result returns stale data** ‚Äî `scrapeRefId` always points to the latest stored result for that pointer. Trigger a fresh crawl to refresh.
